## 🚨 核心安全警告

**本工具设计用于处理高价值资产。在使用前，请务必遵守以下安全协议：**

1. **物理隔离**：必须在**开启飞行模式**、**拔除 SIM 卡**且**彻底断网**的设备上运行。

## 🛡️ 项目理念与技术突破

传统的网页生成器仅依赖 `window.crypto.getRandomValues()`（系统伪随机数生成器）。在刚恢复出厂设置的旧手机上，系统熵池可能尚未收集到足够的随机性，存在理论上的被预测风险。

**HAB-COLD-WALLET 引入了“混合熵 (Mixed Entropy)”机制：**

* **⚡ 物理熵采集**：通过全屏遮罩层，强制要求用户在屏幕上进行无规律的滑动、点击。程序将采集您的 **X/Y 坐标** 及 **毫秒级时间噪声**。
* **🔒 双重哈希算法**：
* `系统熵` = 浏览器底层 CSPRNG 生成
* `用户熵` = SHA256(您刚才的混乱手势数据)
* **`最终种子` = SHA256( 系统熵 + 用户熵 )**


* **🛡️ 供应链纯净**：单文件 HTML，**零外部依赖**。移除了所有 CDN、统计脚本（如 Cloudflare）、字体库请求。

## ✨ 功能特点

* **BIP39 标准**：原生支持生成 12 / 24 位助记词，兼容 Ledger, Trezor, MetaMask 等主流钱包。
* **可视化熵池**：橙色能量条直观展示随机性采集进度，未填满不可生成。
* **反取证设计**：
* 无“复制”按钮，防止剪贴板泄露。
* 无“下载”按钮，防止生成后文件残留。
* 生成后内存变量即刻覆写清理。


* **代码审计友好**：所有逻辑（UI、算法、词表）均包含在一个 HTML 文件中，任何懂 JavaScript 的人均可审计。

## 🚀 使用指南 (标准冷操作流程)

### 第一步：准备代码

1. 下载本项目中的 `index.html`。
2. **校验完整性**：建议自行审查代码，确认无外部网络请求。

### 第二步：物理传输 (Air Gap)

1. 准备一台**永不联网**的旧手机/电脑。
2. 通过 **USB 数据线** 将 HTML 文件传输至设备（严禁使用微信、QQ、网盘传输）。
3. 拔掉数据线，开启飞行模式。

### 第三步：生成助记词

1. 用浏览器打开 `index.html`。
2. 点击“开始生成”，进入**熵采集模式**。
3. 在屏幕上**疯狂滑动手指**或**移动鼠标**，直到进度条填满 100%。
4. 屏幕将显示生成的助记词。

### 第四步：物理备份

1. 使用纸笔抄写助记词（建议抄写两份，异地保存）。
2. **验证**：刷新页面，按同样流程重新生成一次，确保结果完全不同（验证随机性）。
3. **销毁**：关闭页面，删除 HTML 文件，甚至物理销毁设备。

## 🔍 如何验证安全性？

我们鼓励用户不信任我们，而是自行验证：

1. **检查网络**：在浏览器按 F12 打开开发者工具 -> Network 标签，你会发现**没有任何请求**。
2. **检查算法**：搜索代码中的 `generateMnemonic` 函数，查看 SHA-256 混合逻辑。
3. **检查后门**：搜索代码，确认没有 `fetch`, `XMLHttpRequest`, `src="http..."` 等上传代码。
